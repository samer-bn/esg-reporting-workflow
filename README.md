# ESG Reporting Workflow

This project demonstrates a simple Python-based ESG reporting workflow using mock data.

## Objective

The objective is to show how structured data workflows can improve ESG reporting by automating data ingestion, cleaning, validation, analysis, and output generation.

## What the workflow does

- Ingests ESG data from a live source
- Cleans and structures revenue, cost, emissions, and reporting date fields
- Identifies incomplete or inconsistent data
- Separates usable reporting data from data quality issues
- Calculates profit, profit margin, and emissions intensity
- Generates company-level and regional ESG performance summaries
- Exports reporting outputs to Excel

## Key takeaway

The main value of the workflow is not the analysis itself, but the repeatability of the process. A structured workflow allows updated data to be refreshed, cleaned, validated, and analyzed quickly and consistently.
